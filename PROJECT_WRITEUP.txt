ABSTRACT

The AI-Powered Chhattisgarh Travel Planner is a web-based travel recommendation and itinerary generation system developed as a college mini project. The application helps users explore tourist destinations in Chhattisgarh, view destination details, understand different travel experiences, and generate a personalized day-wise itinerary using Google Gemini AI.

The system accepts user inputs such as trip duration, budget, and travel category, then sends these preferences to a backend service. The backend creates a prompt for the AI model, receives the generated response, and returns a structured itinerary to the frontend. The project demonstrates practical use of modern web development, API integration, responsive UI design, and AI-assisted content generation.

1. INTRODUCTION

Travel planning often requires time, research, and decision-making. Many travelers want an itinerary that matches their budget, available days, and travel interests. This project addresses that need by combining a travel discovery interface with an AI-based itinerary generator.

The application focuses on tourist destinations in Chhattisgarh, including temples, waterfalls, heritage sites, and forest attractions. Users can browse places, understand the type of experience each location offers, and generate a custom travel plan suited to their preferences.

This mini project demonstrates how a frontend application can interact with a backend API and an AI service to solve a practical real-world problem.

2. PROBLEM STATEMENT

Planning a trip manually can be difficult because it involves selecting destinations based on interests, balancing time and budget, arranging places in a logical order, identifying suitable travel categories, and creating a day-wise itinerary.

The goal of this project is to simplify this process by providing an AI-powered planner that can generate a structured itinerary quickly.

3. OBJECTIVES

The main objectives of the project are:

- To build an attractive tourism-focused web application.
- To present selected destinations of Chhattisgarh.
- To allow users to explore different travel experiences.
- To generate a personalized itinerary using AI.
- To display itinerary results in a clear and readable format.
- To practice full-stack web application development.

4. SCOPE OF THE PROJECT

The project includes the following features:

- Home page with a visual travel banner and featured destinations.
- Destination detail pages with descriptions and map links.
- Experiences page for selecting travel themes.
- AI itinerary planner form.
- Backend API for itinerary generation.
- Results page for viewing the generated plan.

This project is designed for educational purposes and demonstrates how AI can be applied in a tourism-related application.

6. SYSTEM ARCHITECTURE

The project follows a client-server architecture with three layers:

a) Frontend Layer
The React frontend handles user interaction, navigation, form input, and result display.

b) Backend Layer
The Express backend receives itinerary requests, builds prompts, and forwards them to the AI service.

c) AI Layer
Google Gemini generates the trip itinerary based on budget, duration, and category.

Data Flow:
User Input -> Frontend Request -> Backend Prompt Creation -> Gemini Response -> Frontend Itinerary Display

7. MODULES OF THE PROJECT

Home Module:
Introduces the travel planner with a hero section, destination cards, and a call-to-action to start planning.

Destination Module:
Displays destination-specific pages including title, description, image, and Google Maps link.

Experience Module:
Helps users choose a trip style such as spiritual travel, waterfall visits, heritage exploration, or forest adventures.

Planner Module:
Collects duration, budget, and travel category. Also includes preset options for quicker input.

Results Module:
Displays AI-generated itinerary in day-wise card format.

Backend Module:
Handles route processing, controller logic, service methods, prompt construction, and Gemini API communication.

8. WORKING PRINCIPLE

Step 1: The user opens the website and browses destinations and experiences.
Step 2: The user navigates to the planner page.
Step 3: The user enters duration, budget, and category.
Step 4: The frontend sends the input to the backend API.
Step 5: The backend creates a prompt for Gemini AI.
Step 6: Gemini returns itinerary content in a structured format.
Step 7: The frontend parses and displays the itinerary on the results page.
